【GMS笔记】11.23

2020-11-23
  1. sprite的origin实际上就是sprite的本质点，以👇下图为例，如果origin是（0，0）点，那么当ship旋转是，就是以（0，0）为中心旋转，若origin是（16，16）点，旋转中心点就是（16，16）。action_create instances时，产生的Instance如果使用（0，0）坐标且勾选relative，那么也是以origin点为原点。


  2. 可以设置camera去捕捉game room的一部分区域，以及设置viewport去展示camera捕捉到的画面。viewport决定了游戏窗口的大小。
  3. frames = steps，一帧等于一步，step event实际上就是规定游戏每帧发生的事件，是定义核心逻辑的绝佳场所
  4. create事件仅仅当object产生时运行一次，是定义变量、执行仅需一次的行动的场所
  5. 所有的functions都有特定的作用，但只有部分会return a value。例如keyboard_check返回1/0; instance_create返回一个ID；但move_wrap不返回值。
  6. 当object被摧毁后，附在其上的所有变量都会删除。
  7. 今日成果：平面移动；create instance ; collision
